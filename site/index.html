02 / Method

Boring by design.

A condition should be interesting to the manifest and profoundly boring to the machine.

  1. Native values Foundation preserves booleans, numbers, arrays, and dictionaries as real plist types.
  2. Stock runtime Runs on Apple’s Perl 5.8.8 with no Python, package manager, or added ecosystem.
  3. Safe loading Plugins are ownership-checked, loaded in isolation, and skipped individually when broken.
  4. One clean write Results are collected, sidecar-locked, and atomically serialized to Munki’s conditional items.

03 / Catalog

Small answers.
Useful inventory.

One runner discovers a focused library of non-executable plugins. Each perl returns typed values; none carries its own plist machinery.

01

People & sessions

Admin users, console identity and login state, local home directories, CrashPlan username.

02

Security & management

FileVault, Gatekeeper, SIP, managed user state, Back to My Mac, enabled system extensions.

Physical or virtual, with vendor identification for VMware, VirtualBox, and Parallels.

04 / Quickstart

Install.
Select.
Check in.

The package places the runner and its plugin library at /usr/local/munki/conditions.

Read the complete installation guide →

Install the latest package

sudo /usr/sbin/installer \
  -pkg munki-perls-0.1.N.pkg \
  -target /

Choose only the perls you need

Use managed org.munki.perls preferences, or place the same dictionary in perls/config.plist:

<?xml version="1.0" encoding="UTF-8"?>
<plist version="1.0">
<dict>
<key>included_perls</key>
<array>
    <string>console_user</string>
    <string>virtual_type.pl</string>
</array>
</dict>
</plist>

Managed preferences take precedence. With neither selection source, every installed plugin runs.
